# Redline Cult Rules

Last updated: 2026-05-05

## Mission

Build a high-retention short-form content system for cars and bikes that improves through repeated testing, not guesswork.

## Evidence Status

- Current repo memory is still pre-evidence.
- `memory/analysis/` does not yet contain a weekly review to promote from.
- Treat pattern files as assumptions unless they sit under a confirmed section with repeated support.

## Non-Negotiables

- The first 1.0-1.5 seconds must create immediate motion, tension, or sensory payoff.
- Open with the strongest shot, not with setup.
- Every clip must increase intensity, novelty, or curiosity.
- Videos should feel premium, aggressive, and visually clean.
- Avoid explanation-heavy concepts that slow the opening.
- Optimize for retention first, not information density.

## Content Priorities

1. Hook strength
2. Visual escalation
3. Pacing and cut density
4. Loop quality
5. Topic selection

## Audience Signals To Target

- Speed
- Engine sound
- Mechanical detail
- Status and exclusivity
- Rare or extreme machines
- Clear contrast or comparison

## What To Avoid

- Static openings
- Slow cinematic intros
- Weak descriptive hooks
- Generic edits without a payoff
- Long text overlays
- Clips that do not escalate
- Forcing trends that do not fit the brand

## Hook Rules

- Keep hooks short and aggressive.
- Use curiosity, tension, flex, or disbelief.
- Prefer lines that imply payoff instead of explaining context.
- If the hook sounds like a title, it is too weak.

## Structure Rules

- Ideal duration is short enough to stay dense and replayable.
- Start with action.
- Use fast cuts unless a longer shot clearly increases tension.
- Place the strongest payoff before the end.
- End in a way that visually or sonically loops back into the opening.

## Decision Rules

- Do more of what works at least 2-3 times, not once.
- Do not kill an idea category based on one weak post.
- Separate hook failure from topic failure.
- Separate edit failure from concept failure.
- Separate payoff failure from hook, topic, and edit failure.
- Only promote a pattern to a rule after repeated evidence.

## Weekly Update Rule

At the end of each review cycle:

- Add repeated winners to `memory/patterns/winning-hooks.md` or `memory/patterns/winning-formats.md`
- Add repeated losers to `memory/patterns/failed-patterns.md`
- Add caption and hashtag learnings only when packaging shows repeated impact across multiple posts
- Rewrite this file only when the evidence is clear
